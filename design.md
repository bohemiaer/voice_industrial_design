# Voice Painting Frontend Design

日期：2026-06-12

## 1. 设计结论

本次前端页面方向确定为：

- 页面结构采用 `A 画布主导`
- 视觉风格采用 `A2 Precision Lab`
- 视觉参考吸收所给截图中的轻量创作工作台语言

页面应呈现为一个高完成度、轻盈、安静、精密的创作工作台，而不是厚重后台或强科技感实验界面。

## 2. 整体目标

这个页面的核心任务不是展示很多功能，而是让用户一进入就立刻理解：

1. 左侧是概念树画布，是整个产品的主空间。
2. 右侧是语音对话与状态区，是辅助理解和执行的控制台。
3. 用户通过语音推动树状发散，AI 的理解、确认和反馈围绕这个主循环服务。

页面需要同时满足三点：

1. 让树状分支足够清楚。
2. 让语音交互状态足够明确。
3. 让整体视觉足够轻，不压迫创作感。

## 3. 设计关键词

### 3.1 整体气质

- 克制
- 轻盈
- 安静
- 精密
- 亲和
- 高级但不过度炫技

它更像一个面向设计师的消费级创作工具，而不是企业管理后台，也不是重工业仿真软件。

### 3.2 色彩关键词

- 高明度暖白或微灰白作为主背景
- 冷灰蓝作为次级结构色
- 低饱和中性色作为描边和分隔
- 少量功能色用于状态反馈

推荐倾向：

- 主背景：偏暖白、浅灰白
- 面板底色：白色或极浅灰
- 描边：浅灰蓝、浅灰
- 强调色：低饱和蓝灰或中性绿色
- 警示色：尽量克制，只在失败、冲突、取消时使用

避免高饱和蓝紫、霓虹渐变、厚重深色背景。

## 4. 版式结构

### 4.1 总体布局

页面采用画布主导的双区结构：

- 左侧 `75%`：概念树画布区
- 右侧 `25%`：对话与状态区

右侧对话区支持收起。收起后不应完全消失，而应保留一条窄竖向状态条，继续承载：

1. 录音入口
2. 当前目标节点摘要
3. 当前任务状态
4. 展开按钮

这样可以避免主链路丢失，同时保证大画布场景下的沉浸感。

### 4.2 工作台外框

参考图中的整体工作台语言，页面不建议做成完全铺满浏览器的硬边应用，而建议采用：

- 大圆角主工作台容器
- 轻描边
- 极浅阴影
- 外层留白

整体观感像一张被放在桌面上的精密创作面板。

### 4.3 顶部工具区

顶部工具应采用轻量悬浮胶囊，而不是厚导航栏。

建议保留：

1. 缩放控制
2. 回到当前节点
3. 撤销
4. 重做
5. 导出

工具数量必须克制，避免把顶部做成功能堆叠区。

## 5. 左侧画布区

### 5.1 角色定位

左侧画布区是绝对主角，用于承载概念树的生长、分支关系和当前探索位置。

它需要优先表达：

1. 当前树结构
2. 当前选中节点
3. 生成中的分支状态
4. 当前层和路径关系

### 5.2 视觉表现

画布区域应保持大量留白，背景尽量干净，不引入复杂纹理和强装饰。

可接受的背景处理：

- 极浅暖灰白
- 非常轻的网格或定位参考
- 接近不可见的层级阴影

不建议使用明显的工程网格、重线框、科技 HUD 风格元素。

### 5.3 节点布局原则

节点排布要优先保证“树结构关系清楚”，其次才是单个节点卡片的装饰性。

原则：

1. 根节点和当前活跃路径必须容易识别
2. 同层节点之间要有稳定节奏
3. 子分支展开要尽量自然，不挤压主画布可读性
4. 连线表达应轻，不应喧宾夺主

## 6. 右侧对话区

### 6.1 角色定位

右侧区域不是传统聊天窗口，而是“语音脑暴控制台”。

它的职责是：

1. 展示语音转写
2. 展示 AI 对本轮意图的理解
3. 展示确认和纠错状态
4. 提供继续发散提示
5. 承担录音入口

### 6.2 宽度与状态

- 展开宽度占整体约 `25%`
- 支持收回
- 收回后保留窄状态条

展开态下建议的纵向结构：

1. 顶部：当前会话与当前目标摘要
2. 中部：消息流、系统理解、确认状态
3. 底部：语音输入入口

### 6.3 信息密度

右侧区应保持中低密度，不能做成高密度文本面板。

更适合的方式是：

- 短消息块
- 状态卡片
- 单轮操作摘要
- 少量提示

避免长篇对话堆叠和大段说明文案。

## 7. 卡片与节点风格

### 7.1 总体卡片语言

卡片风格应继承参考图里的轻盈面板语言：

- 大圆角
- 薄描边
- 浅阴影
- 轻微浮起
- 内容留白充分

卡片之间的区分主要依靠层级、留白和结构，而不是大面积色块。

### 7.2 节点卡片

节点卡片建议定义为“轻量作品卡”，而不是便签或列表项。

每个节点可包含：

1. 草图缩略图
2. 方向名称
3. 层内序号
4. 一句话解释

信息优先级：

1. 图像
2. 名称
3. 当前状态
4. 解释

### 7.3 选中与状态反馈

当前选中节点不建议用高饱和背景高亮，而建议使用：

- 更清晰的描边
- 轻微外发光或阴影提升
- 稳定、克制的强调色

生成中、待确认、失败等状态也应以轻方式表达：

- 角标
- 边框变化
- 微动效
- 小型状态点

避免大片红黄蓝强提示污染整个画面。

## 8. 视觉语言提炼

基于参考图，可提炼出以下页面语法：

### 8.1 可复用特征

- 大工作台容器
- 高明度背景
- 中心内容强聚焦
- 左右面板都很轻
- 悬浮胶囊工具条
- 圆角和细描边统一
- 控件精致但存在感克制

### 8.2 迁移到本产品时的重点

不能直接复制参考图的三栏编辑器结构，而要把它转译成：

- 中心：概念树画布
- 右侧：对话控制台
- 顶部：悬浮操作胶囊
- 底部或右侧下部：语音输入主入口

我们的重点不是“编辑单一对象”，而是“沿树状结构持续发散多个方向”。

## 9. 交互气质

交互反馈应遵循以下原则：

- 轻
- 短
- 稳
- 不打断

建议：

1. 录音态有明确但不刺眼的视觉反馈
2. AI 理解完成后，以一张操作卡片方式出现
3. 高风险操作确认时，界面进入轻确认态，而不是弹出传统警告框
4. 节点生成结果逐步落位，形成“树在长出来”的感觉

## 10. 避免事项

以下方向应避免：

1. 深色大面积 UI
2. 强科技蓝紫风格
3. 过重的工业硬核感
4. 过多玻璃拟态和发光效果
5. 复杂三栏四栏后台布局
6. 把右侧区做成传统 IM 聊天软件
7. 节点卡片套节点卡片的高嵌套结构
8. 过强的参数面板感

## 11. 推荐落地方案

最终推荐版本：

- 结构：`画布主导双栏布局`
- 比例：`左 75% / 右 25%`
- 风格：`Precision Lab`
- 气质：`轻盈精密的设计师创作工作台`
- 对话区：`可收起，收起后保留窄状态条`
- 组件语言：`大圆角、薄描边、浅阴影、悬浮胶囊工具`

这套设计能同时满足：

1. 树状脑暴逻辑清晰
2. 语音交互状态明确
3. 整体界面有现代创作工具质感
4. 不会因为过度装饰而牺牲可用性

## 12. 后续页面落地建议

如果继续细化前端设计，下一步建议按以下顺序展开：

1. 首页工作台线框
2. 右侧对话区展开 / 收起态
3. 节点卡片样式规范
4. 状态系统规范
5. 录音输入组件规范
6. 画布缩放与当前路径提示规范
