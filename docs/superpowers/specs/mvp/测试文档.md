# Voice Industrial Design 测试文档

日期：2026-06-15

## 1. 测试目标

验证当前 MVP 是否已经切换到“无确认直出”的语音脑暴链路，并满足以下核心闭环：

1. 用户创建会话后可以直接进入工作台。
2. 每轮语音或文本输入会直接执行，不再进入确认态。
3. 系统支持首层扩展、节点下钻、当前层刷新、跨层继续扩展与单次撤回。
4. 若本轮未提节点，则沿用上一轮实际执行目标节点。
5. 若本轮未提数量，则默认生成 `3` 个方向。
6. 前端工作台、后端 API、shared schema 与 DeepSeek 官方 chat、SiliconFlow ASR / 出图链路行为一致。

## 1.1 当前统一口径（2026-06-16）

本测试文档当前以以下能力为主验收口径：

1. `diverge`
2. `refresh`
3. `delete`
4. `undo`
5. `redo`

其中：

1. `refresh` 固定为“刷新当前节点下最近一次生成出来的那组子节点”。
2. `undo` 撤回最近一次 `diverge / refresh / delete`。
3. `redo` 恢复最近一次被 `undo` 的树操作。
4. 任意生成任务运行中时，发送按钮置灰，无法再次发送消息。

## 2. 测试规则

### 2.1 节点规则

1. 描述里明确提到节点时，使用描述中的节点。
2. 描述里没有提到节点时，沿用上一轮实际执行目标节点。
3. 当前会话还没有任何已执行树操作时，默认使用根节点。

### 2.2 数量规则

1. 描述里明确提到数量时，按描述数量执行。
2. 描述里没有提到数量时，默认生成 `3` 个方向。
3. 数量仍受系统上限约束。

### 2.3 撤回规则

1. 仅支持撤回最近一次已执行树操作。
2. 撤回不需要二次确认。
3. 撤回后树恢复到上一轮执行前的状态。

## 5. 后端 API 回归

### TC-API-001 Session API

1. `POST /api/sessions` 返回 `201`。
2. `GET /api/sessions/{sessionId}/tree` 返回空树。
3. `GET /api/sessions/{sessionId}/messages` 返回空消息。

### TC-API-002 Voice Turn API

1. 文本 voice turn 直接执行并返回 `completed task`。
2. multipart 音频 voice turn 能正确读取 `audio` 与 `mimeType`。
3. 未传 `audio` 且未传 `transcriptText` 时返回 `VOICE_TURN_INPUT_REQUIRED`。

### TC-API-003 默认规则

1. 未明确指定节点时，沿用上一轮实际执行目标节点。
2. 未明确指定数量时，默认生成 `3` 个方向。
3. 明确写出数量时，按显式数量执行。

### TC-API-004 树操作

1. `expand_branches` 直接生成首层节点。
2. `branch_deeper` 直接在目标节点下新增子层。
3. `refresh_layer` 直接替换当前层。
4. `undo` 恢复最近一次已执行树操作。

## 6. Agent 网关回归

### TC-AGENT-001 ASR

1. 请求发送到 `/audio/transcriptions`。
2. 使用 `FunAudioLLM/SenseVoiceSmall`。
3. 返回 `transcriptText`。

### TC-AGENT-002 Brainstorm Assistant

1. 请求发送到 `/chat/completions`。
2. 使用 `deepseek-ai/DeepSeek-V4-Flash`。
3. 输出必须通过 shared schema。
4. 当模型返回 `rewrittenIntentForConfirmation: null` 时，解析层仍能兼容。

### TC-AGENT-003 图像生成

1. 请求发送到 `/images/generations`。
2. 使用 `Tongyi-MAI/Z-Image-Turbo`。
3. 返回有效 `imageUrl`。

## 7. 前端工作台回归

1. 页面只保留画布、消息流、录音输入主链路。
2. 不再依赖 `ConfirmationCard`、确认按钮或确认文本指令。
3. 用户提交文本后立即插入 optimistic user message。
4. 任务完成后从真实 API 回刷树、消息、任务状态。
5. 节点选中会直接更新当前目标节点。
6. 撤回时前端会把最后一条可撤回 `TreeOperation` 回传给 API。

## 9. 验收标准

1. 主链路不再出现 `awaiting_confirmation`。
2. 默认节点规则与默认数量规则在自动化测试和手工 E2E 中一致。
3. 六轮主链路可以按顺序执行并得到正确树状态。
4. 撤回不会再清空整张画布。
5. 自动化测试覆盖 DeepSeek 官方 chat 请求格式，以及 SiliconFlow ASR / 出图请求格式。

## 10. 当前风险

1. 本机若未配置 `DEEPSEEK_API_KEY`，chat 会退回到 SiliconFlow 兼容链路，真实联调结果会受回退稳定性影响。
2. `confirm` / `cancel` 接口仍为兼容保留，不属于主产品链路。
3. 当前文档只将“无确认直出”作为主验收口径，旧确认流程不再作为通过标准。
